# 🚀 الدليل الشامل لرفع وتحديث المشروع (GitHub, Vercel, Firebase & HuggingFace)

يحتوي هذا الملف على الخطوات الدقيقة والصحيحة لرفع التحديثات البرمجية الأخيرة وتجنب حدوث أي أخطاء أثناء البناء (Build) على خوادم الاستضافة.

---

## 📌 ملخص التحديثات المضافة في هذا الإصدار:
1. **حل خطأ البناء (Build Error)**: تم تعديل خطاف الـ `postinstall` في `package.json` ليتفحص وجود ملف الباتش قبل تشغيله لعدم التسبب في انهيار الحاوية (Container Build Crash).
2. **تفعيل نظام الصداقة الكامل**: تفعيل واجهات البحث وقبول/رفض وحظر طلبات الصداقة وإدارة المحظورين في `ProfileModal.tsx`.
3. **تحديث بروفايل الآخرين**: ربط أزرار الحظر/إلغاء الحظر والخصوصية وقمع التفاعل التلقائي في `UserProfileModal.tsx`.
4. **تأمين Firestore**: قواعد أمان صارمة في `firestore.rules` تمنع إرسال طلبات للمستخدمين المحظورين.

---

## 1️⃣ الرفع والتحديث على GitHub و Vercel (الواجهة الأمامية)

بمجرد رفع التحديثات إلى فرع `main` على GitHub، سيقوم **Vercel** تلقائياً بسحب التحديثات وإعادة بناء الموقع بدون أي تدخل منك.

### الخطوات (نفذها في الـ Command Prompt / PowerShell الخاص بجهازك):
```bash
# 1. تأكد من أنك على الفرع الرئيسي
git checkout main

# 2. إضافة جميع التعديلات الأخيرة
git add .

# 3. تسجيل التعديلات برقم إصدار ووصف
git commit -m "fix: resolve build error and enable social and blocking UI"

# 4. الرفع إلى مستودع GitHub (Vercel سيبني تلقائياً)
git push origin main
```

---

## 2️⃣ رفع وتحديث قواعد الأمان على Firebase (Firestore Rules)

قواعد الأمان الخاصة بالصداقة والمحادثات والحظر تم كتابتها محلياً في ملف `firestore.rules` ويجب دفعها لحساب Firebase الخاص بك لتطبيقها فورياً.

### الخطوات:
1. افتح مبدل الأوامر في مجلد المشروع وسجل الدخول لـ Firebase (إذا لم تكن مسجلاً):
   ```bash
   firebase login
   ```
2. قم برفع قواعد الحماية فقط دون بقية الخدمات:
   ```bash
   firebase deploy --only firestore:rules
   ```

---

## 3️⃣ الرفع والتحديث على HuggingFace Space (خادم رندر الفيديوهات)

خادم رندر الفيديوهات معزول في فرع مستقل باسم `hf-space` على جهازك، ويقوم بتقديم الخدمة على مساحة HuggingFace عبر بناء حاوية Docker من ملف الـ `Dockerfile`.
بما أننا قمنا بإصلاح مشكلة الـ `postinstall` في ملف `package.json` على الفرع الرئيسي (`main`)، **يجب دمج هذا التعديل في فرع `hf-space` أولاً ثم دفعه لـ HuggingFace** لكي يكتمل البناء بنجاح هناك.

### الخطوات:
```bash
# 1. الانتقال إلى فرع خادم الرندر المستقل
git checkout hf-space

# 2. دمج التعديلات الأخيرة من الفرع الرئيسي (لتضمين إصلاح postinstall في package.json)
git merge main -m "merge: sync main changes into hf-space to fix build"

# (ملاحظة: في حال حدوث أي تعارض (Conflict)، قم بحفظ الملفات ثم كتابة git add . و git commit)

# 3. رفع الفرع المحلي hf-space إلى الفرع الرئيسي (main) في مساحة HuggingFace
git push space hf-space:main --force
```
