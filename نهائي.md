# 📖 الدليل النهائي والشامل لمشروع تطبيق القرآن الكريم

هذا المستند هو المرجع الوحيد والكامل لك لإدارة وفهم مشروعك (الموقع والتطبيق) بعد انتهاء عملية التطوير بنجاح.

---

## 🏗️ 1. هيكلة المشروع (وين الكود؟)

المشروع مبني باستخدام **Next.js 16** مع **Capacitor** للأندرويد.

- **`src/app/`**: يحتوي على الصفحات الرئيسية للموقع (المصحف، الأذكار، مواقيت الصلاة).
- **`src/components/`**: المكونات البرمجية (مثل `PrayerTimes.tsx` للأذان، و `RenderModal.tsx` لاستوديو الفيديو).
- **`public/`**: الملفات العامة (الصور، اللوجو، ملفات الصوت الصغيرة، والـ `sw.js` المسؤول عن العمل بدون إنترنت).
- **`android/`**: كود تطبيق الأندرويد الكامل.
- **`download-site/`**: صفحة التحميل الاحترافية التي صممناها لـ Netlify.

## 📱 3. تطبيق الأندرويد (APK)

تطبيق الأندرويد هو نسخة "ثابتة" (Static Export) من الموقع يتم تغليفها داخل Capacitor.

- **ميزة الـ Offline:** بمجرد أن يستمع المستخدم لسورة، يتم حفظها تلقائياً في الهاتف بفضل الـ Service Worker.
- **الأيقونة:** يتم تحديثها تلقائياً من ملف `logo.png` الموجود في مجلد `public/logo`.
- **رابط التحميل:** التطبيق مبرمج ليشير دائماً إلى `latest release` على GitHub، مما يعني أنك بمجرد رفع APK جديد، سيحصل الجميع عليه.

---

## 🛠️ 4. الأوامر السحرية (كيف تدير المشروع؟)

لقد اختصرت لك كل العمليات المعقدة في أوامر بسيطة تنفذها في التيرمينال:

| الأمر | وظيفته |
| :--- | :--- |
| `npm run final-build` | **الأمر الأهم:** يقوم بتحديث الأيقونات + بناء الـ Offline + إنتاج الـ APK النهائي. |
| `git add . && git commit -m "update" && git push` | لرفع أي تعديل جديد قمت به إلى GitHub وتحديث الموقع. |
| `node update-assets.js` | لتحديث أيقونات الأندرويد فقط إذا قمت بتغيير صورة اللوجو. |

---

## 🎥 5. استوديو الفيديو (Rendering)

يعمل الاستوديو بنظامين:
1. **Browser Render (المتصفح):** سريع جداً ولكنه ينتج ملفات WebM (لا تعمل على تيك توك أحياناً).
2. **Server Render (السيرفر):** يرسل البيانات لسيرفر Hugging Face وينتج ملف MP4 احترافي.
   - **تنبيه:** إذا توقف السيرفر، يجب الدخول للرابط الخاص به في المتصفح "لإيقاظه" (Wake up).

---

## 🔒 6. ملاحظات للأمان والاستقرار

- **ملف الـ Lockfile:** احرص دائماً على عدم وجود ملف `package-lock.json` في مجلدات خارج المشروع (مثل مجلد المستخدم الرئيسي) لتجنب بطء البناء.
- **تحديث السور:** يتم جلب السور من API خارجي موثوق، ولكن بمجرد التشغيل يتم تخزينها مؤقتاً لتعمل بدون إنترنت.
