# 🕋 الدليل الشامل لإدارة وتطوير تطبيق القرآن (Master Guide)

هذا الملف هو "عقل" المشروع، يحتوي على شرح لكل جزء وإزاي تتعامل معاه. أي تعديل بنعمله هيتم توثيقه هنا.

---

## 🛠️ 1. هيكل المشروع (Where is everything?)

- **الموقع الرئيسي:** `src/app` و `src/components` (مبني بـ Next.js 16).
- **تطبيق الأندرويد:** مجلد `android` (مبني بـ Capacitor).
- **محرك الفيديوهات:** ملفات `render.mjs` و `render-server.mjs` (مبني بـ Remotion).
- **صفحة التحميل:** مجلد `download-site`.

---

## 🚀 2. إزاي ترفع تحديث جديد؟

### للـموقع (Vercel):
1. افتح الـ Terminal.
2. اكتب الأوامر دي بالترتيب:
   ```bash
   git add .
   git commit -m "تعديل جديد: وصف التعديل"
   git push origin main
   ```
3. فيرسل هيحدث الموقع لوحده في دقيقة.

### للـتطبيق (APK):
1. بعد ما تعدل الكود، شغل الأمر ده عشان يجهز الملفات للأندرويد:
   ```bash
   npm run cap:sync
   ```
2. افتح **Android Studio**.
3. من قائمة `Build` اختار `Build Bundle(s) / APK(s)` ثم `Build APK(s)`.
4. الملف النهائي هيكون في مسار: `android/app/build/outputs/apk/debug/app-debug.apk`.

---

## 🎨 3. تغيير اللوجو والأيقونات

1. حط صورتك الجديدة في `public/logo/logo.png`.
2. شغل سكربت التحديث الشامل:
   ```bash
   node do-everything.js
   ```
   *هذا السكربت هيوزع اللوجو على الـ PWA، صفحة التحميل، وأيقونات الأندرويد أوتوماتيكياً.*

---

## ⚠️ 4. حل المشاكل الشائعة

### مشكلة الخروج المفاجئ (App Crash):
- **السبب:** غالباً بسبب اختلاف اسم الـ Package في ملف `capacitor.config.ts` عن الملفات الفعلية في `android/app/src/main/java`.
- **الحل:** تم ضبط المسارات لتعمل مع `quran1mu.vercel.app`.

### مشكلة الرندرة (Concurrency Error):
- **السبب:** السيرفر بيحاول يشغل أكتر من عملية رندر في نفس الوقت والنظام مش بيسمح.
- **الحل:** تم تثبيت `concurrency: 1` في السكربتات لضمان الاستقرار.
